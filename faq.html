Q: What is the main goal of this project?
A: To build a serverless ride-request web application using AWS services like Cognito, Lambda, API Gateway, DynamoDB, and Amplify.
Q: How is user authentication handled in this project?
A: User authentication is managed via Amazon Cognito, which supports sign-up, login, and email verification.
Q: What happens when a user requests a ride?
A: The frontend sends a request to API Gateway, which invokes a Lambda function that selects a unicorn and stores ride data in DynamoDB.
Q: How is the backend deployed and updated?
A: The backend is deployed using AWS Amplify, which automatically rebuilds and redeploys the app on every GitHub commit.
Q: What programming language is used for the backend logic?
A: The backend Lambda function is written in Node.js (JavaScript).
Q: Is the frontend integrated with the backend?
A: Yes, the frontend (HTML/CSS/JS) communicates with the backend via API Gateway using JWT tokens for secure interaction.
Q: What database is used to store ride information?
A: Ride data is stored in Amazon DynamoDB, a fully managed NoSQL database service.
